Builder path

From docs context to a safe Arc testnet prototype.

The app now shows the intended implementation sequence instead of leaving the demo as a loose mockup. Each step keeps private keys and autonomous spending out of scope until the testnet flow is reviewed.

1 · Ground the agentConnect an AI coding tool to Arc MCP, then ask it to cite the docs it used before scaffolding code.
2 · Create intent dataRepresent amount, asset, recipient, memo, expiry, and status as reviewable JSON before any wallet action.
3 · Require human approvalKeep signing manual. The agent can prepare context, but the user confirms every transaction.
4 · Observe and documentTrack status through ArcScan or app logs, then publish the findings back into the docs and prompts.

Safety default: v0 is non-custodial and review-first. No backend custody, no private-key handling, and no autonomous mainnet spending.
