# x402 MCP manifest

The local x402 challenge server now exposes a machine-readable, MCP-style manifest for the paid-agent boundary. It keeps the demo discoverable by agents without adding wallet signing, private keys, or live settlement.

Open the local server:

```bash
python3 examples/x402-local-challenge-server/server.py --port 8087
```

Then inspect the protected endpoint:

```bash
curl -s http://127.0.0.1:8087/protected
```

The unpaid `402 Payment Required` response includes:

- `id`: deterministic local challenge id;
- `accepts`: x402-shaped payment terms for Arc Testnet USDC;
- `unitEconomics`: integer `priceMicroUsd` plus display price;
- `mcpManifest`: agent-readable metadata, tool schemas, safety flags, and production replacement boundary.

## Manifest shape

The manifest is generated by `build_mcp_manifest` in `examples/x402-local-challenge-server/server.py`.

Key fields:

- `name`: `arc-local-x402-paid-agent`;
- `network`: Arc Testnet constants, including chain ID `5042002 / 0x4cef52`;
- `payment`: local demo asset, amount, pay-to address, and resource name;
- `unitEconomics`: USDC pricing stored as integer microUSD to avoid float ambiguity;
- `safety`: explicit booleans for testnet-only, human approval, no private keys, no autonomous spending, and no transaction broadcast;
- `tools`: MCP-style tool descriptions for inspecting the challenge and retrieving the paid resource after a local demo proof;
- `builderContext`: separated facts, repo choices, assumptions, and non-goals.

## Tool list

The local manifest intentionally exposes only two tools:

- `inspect_payment_challenge` — returns the current local challenge without requiring a payment proof.
- `get_paid_resource` — returns the protected local report only when `X-Payment` carries the accepted demo proof.

This is a discovery contract for future agents. It is not yet a full stdio MCP server and does not call Circle Gateway.

## Safe paid-agent flow

1. Client requests `/protected`.
2. Server returns `402` with the challenge and `mcpManifest`.
3. Human reviews the resource, amount, recipient, and safety flags.
4. Demo client sends `X-Payment: local-demo:<challenge-id>:0.01`.
5. Server returns `200` with the paid payload, receipt, unit economics, and manifest metadata.

No funds move in this flow.

## Production replacement boundary

Replace `LocalDemoVerifier` with Circle Gateway/x402 verification only after these rules are explicit and tested:

- exact Arc Testnet network and USDC asset gates;
- pay-to address ownership and rotation policy;
- proof format, expiry, replay protection, and nonce storage;
- settlement finality requirements;
- logging and privacy policy for payment proofs;
- verifier outage behavior;
- mandatory human approval before any wallet action.

Until those gates exist, the local proof remains a deterministic demo switch, not a payment credential or settlement proof.
